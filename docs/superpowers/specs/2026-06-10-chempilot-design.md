# ChemPilot 智能化学实验数字平台设计说明

## 1. 项目定位

ChemPilot 是一个面向中学化学课堂实验场景的学生端 AI 实验智能体原型。项目由化学与材料专业基础、应用化学方向背景出发，把中学化学实验中的“现象观察、原理解释、错误纠正、实验结论形成”做成可交互、可解释、可复现的数字化学习流程。

第一版不追求完整双端平台，而是先完成一条清晰的学生学习闭环：

学生输入或选择实验条件 -> 系统识别反应类型 -> 展示实验现象 -> 判断学生预期是否合理 -> AI 智能体生成解释 -> 输出方程式、结论、安全提醒和追问。

这个范围来自现有项目材料中的“先做一条线”要求，目标是保证作品可演示、可说明、可复现，同时保留教师端和数据分析作为后续拓展。

## 2. 大赛适配目标

项目围绕“AI+未来基础教育教师创新”展开，重点不是让 AI 直接替学生完成答案，而是让 AI 支持学生完成实验学习中的关键认知动作：

- 观察：看见实验现象，而不是只背结论。
- 解释：把气泡、颜色变化、沉淀、放热等现象连接到化学原理。
- 纠错：识别学生对反应现象的错误预期。
- 总结：形成规范实验结论和化学方程式。
- 安全：提醒中学实验操作中的风险和规范。

作品报告中可强调：应用化学专业学生将专业知识转化为基础教育数字化工具，帮助中学生在实验资源不足、课堂时间有限、观察记录不充分、现象解释困难等场景下获得更稳定的实验学习体验。

## 3. 最小闭环范围

第一版只实现学生端主线，包含以下页面和功能：

1. 实验选择
   - 学生从 4 个代表性中学实验中选择。
   - 也可以在输入框中输入反应或实验描述。

2. 学生预期输入
   - 学生填写“我认为会出现什么现象”。
   - 系统用该输入触发纠错和解释。

3. 实验现象展示
   - 用卡片、步骤条、简洁动画或可视化状态展示实验过程。
   - 每个实验展示关键现象：气泡、颜色变化、沉淀、复燃、浑浊等。

4. 智能体解释
   - 输出现象解释、化学方程式、实验结论、安全提醒、易错点。
   - 若学生预期错误，先温和指出，再解释为什么。

5. 追问与反思
   - 每次实验后生成 1 个课堂追问。
   - 追问用于体现启发式教学，而不是只给静态答案。

## 4. 首批实验库

### 4.1 二氧化碳的制取与检验

推荐作为主展示实验。

- 典型条件：碳酸钙或碳酸钠与稀盐酸反应。
- 关键现象：产生气泡；生成气体通入澄清石灰水后变浑浊。
- 核心原理：碳酸盐与酸反应生成二氧化碳；二氧化碳与氢氧化钙反应生成碳酸钙沉淀。
- 展示优势：现象明显，适合课堂演示，能体现气体制备、检验和结论推理。

### 4.2 盐酸与氢氧化钠中和

推荐作为纠错型实验。

- 典型条件：稀盐酸与氢氧化钠溶液反应，可加入酚酞或温度变化观察。
- 关键现象：通常无气泡；若使用酚酞，颜色随酸碱性变化；中和过程可伴随放热。
- 核心原理：氢离子与氢氧根离子结合生成水。
- 展示优势：适合纠正“酸碱反应都会冒泡”的错误认识。

### 4.3 铁与硫酸铜反应

推荐作为材料与金属活动性关联实验。

- 典型条件：铁钉放入硫酸铜溶液。
- 关键现象：铁表面析出红色铜，蓝色溶液逐渐变浅。
- 核心原理：铁的金属活动性强于铜，发生置换反应。
- 展示优势：与化学与材料、应用化学背景自然衔接，现象具有可视性。

### 4.4 过氧化氢制取氧气

推荐作为气体制备与催化实验。

- 典型条件：过氧化氢溶液在二氧化锰催化下分解。
- 关键现象：产生大量气泡，带火星木条复燃。
- 核心原理：二氧化锰作为催化剂加快过氧化氢分解，生成水和氧气。
- 展示优势：可讲催化剂、气体检验和实验安全。

## 5. 技术架构

第一版采用前端优先的 Web App，便于快速开发、演示和部署。

### 5.1 前端

- 技术栈：React + TypeScript + Vite。
- 主要模块：
  - 实验选择面板
  - 反应输入框
  - 学生预期输入区
  - 实验过程展示区
  - AI 解释结果区
  - 实验记录区

### 5.2 实验规则库

实验规则库使用结构化数据保存，每个实验包含：

- 实验名称
- 适用关键词
- 反应物与条件
- 实验步骤
- 标准现象
- 化学方程式
- 实验结论
- 安全提醒
- 常见误区
- 追问模板

规则库是可复现核心：即使没有真实大模型，也能稳定输出正确实验现象。

### 5.3 智能体解释层

智能体解释层不直接决定实验事实，而是在规则库结果基础上生成适合中学生理解的反馈。第一版可以采用本地模板智能体，后续再接入 OpenAI、DeepSeek、通义千问等模型。

解释层职责：

- 理解学生输入命中的实验类型。
- 对比学生预期与标准现象。
- 生成分层解释：现象 -> 原理 -> 方程式 -> 结论。
- 生成安全提醒和课堂追问。

这样可以避免大模型幻觉，同时满足附件 3 中“AI 模型应用与复现说明”的要求。

## 6. 数据流

1. 学生选择实验或输入反应描述。
2. 系统使用关键词和规则库匹配实验。
3. 学生填写预期现象。
4. 规则引擎返回标准实验对象。
5. 预期判断器比较学生输入与标准现象、常见误区。
6. 智能体解释层生成反馈。
7. 页面展示实验现象、解释、结论和追问。
8. 实验记录保存到本地状态，作为展示学生学习轨迹的证据。

## 7. 用户体验设计

首屏直接进入实验智能体，不做宣传式落地页。

界面应当像一个课堂实验工作台，而不是聊天软件。建议布局：

- 左侧：实验选择和学生输入。
- 中间：实验过程与现象展示。
- 右侧：AI 实验导师反馈。
- 底部或侧栏：本次实验记录。

视觉风格应体现化学课堂和实验室秩序：清晰、克制、可读，避免过度科幻。实验现象可以用轻量动画增强展示性，例如气泡上升、溶液颜色变化、石灰水浑浊、木条复燃提示。

## 8. 暂不纳入第一版的内容

以下内容作为作品报告中的平台拓展，不进入第一版最小闭环：

- 教师登录、班级管理、任务发布。
- 学生账号系统。
- 真实数据库和云端同步。
- AI 自动生成完整实验演示视频。
- 多轮自由聊天式问答。
- 复杂 3D/VR 实验室。

这些方向有价值，但会分散第一版演示重点。第一版先把学生端实验认知闭环做扎实。

## 9. 可复现说明思路

附件 3 可按以下方式填写：

- AI 核心任务：文本处理、诊断评估、创意生成、自动化。
- 模型/工具：
  - React + TypeScript：交互式实验平台。
  - 化学实验规则库：提供可复现实验事实。
  - AI 解释模板或大语言模型：生成面向学生的分层解释和追问。
  - 可选本地数据存储：保存实验记录。
- 标准复现步骤：
  1. 打开平台。
  2. 选择“二氧化碳的制取与检验”。
  3. 输入学生预期，例如“会产生气泡，石灰水变浑浊”。
  4. 点击生成实验反馈。
  5. 查看现象、方程式、结论、安全提醒和追问。

## 10. 成功标准

第一版完成后应满足：

- 能在浏览器中完整演示 4 个中学化学实验。
- 学生输入错误预期时，系统能给出合理纠错。
- 每个实验都有现象、原理、方程式、结论、安全提醒和追问。
- 无需外部服务也能运行核心流程。
- 若配置模型 API，可增强解释表达，但不影响基础复现。
- 界面适合比赛展示，能让评委在 1-2 分钟内看懂项目价值。
