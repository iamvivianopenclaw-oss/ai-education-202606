# ChemPilot V2 设计说明：视觉化实验与记忆巩固闭环

## 1. V2 目标

V2 的目标是让 ChemPilot 从“能解释实验”升级为“能看见实验、记住实验、反复巩固实验”。

第一版已经完成学生端最小闭环：选择实验、输入预期、获得智能体解释、形成学习记录。第二版不改变这个主线，而是在主线中增强两个能力：

- 实验更视觉化：学生能在页面中直观看到实验装置、步骤推进和关键现象变化。
- 帮助记忆避免遗忘：学生做完实验后，系统自动把知识点沉淀成记忆卡片和复习任务，让实验学习能够被回看、回忆和巩固。

V2 仍然面向中学化学课堂实验场景，强调展示性、复现性和教育价值。技术上优先采用 React + TypeScript + CSS 动画，不引入复杂 3D、视频生成或后端数据库，保证比赛现场稳定运行。

## 2. 设计原则

1. 视觉化服务于实验理解

   页面动画要表现化学实验的关键证据，例如气泡、浑浊、颜色变化、红色铜析出、带火星木条复燃。动画不是装饰，而是帮助学生把“观察到什么”与“说明了什么”连接起来。

2. 记忆巩固服务于长期学习

   复习模块不做应试题海，而是围绕现象、方程式、原理、易错点和安全点生成轻量记忆卡片。学生可以遮住答案回忆，也可以做简单判断和短答，逐步减少遗忘。

3. AI 解释与规则事实分离

   实验事实仍由规则库提供，反馈和记忆提示由智能体模板组织语言。这样能保持化学事实准确，也便于后续接入真实大模型。

4. 本地优先，易于复现

   V2 继续保持无后端、无外部 API 的基本演示能力。学习记录和复习状态先保存在浏览器本地状态或 localStorage，便于比赛现场快速复现。

## 3. 实验视觉化设计

### 3.1 新增实验视觉模型

在现有实验规则库基础上，为每个实验增加可视化配置：

- 实验装置类型：试管、烧杯、导管、石灰水、木条、金属片等。
- 初始状态：液体颜色、试剂标签、固体/金属位置。
- 过程状态：气泡、颜色渐变、沉淀/浑浊、析出物、复燃效果。
- 关键观察点：每一步应该让学生注意的现象。
- 现象解释标签：把视觉现象和知识点连接起来，例如“气泡：产生 CO2”“浑浊：生成 CaCO3 沉淀”。

### 3.2 实验时间线

实验舞台从静态卡片升级为“步骤时间线”：

1. 准备试剂
2. 开始反应
3. 观察现象
4. 检验气体或判断产物
5. 得出结论

学生可以点击步骤，也可以使用“播放/暂停/重播”控制实验演示。每个步骤对应一组视觉状态和一句观察提示。

### 3.3 四个实验的视觉重点

- 二氧化碳的制取与检验：碳酸盐与稀盐酸反应产生气泡，气体通入澄清石灰水，石灰水逐渐变浑浊。
- 盐酸与氢氧化钠中和：强调通常无气泡，使用酚酞时颜色随酸碱性变化，中和可伴随温度变化。
- 铁与硫酸铜反应：铁表面出现红色铜，蓝色溶液逐渐变浅。
- 过氧化氢制取氧气：二氧化锰催化后产生大量气泡，带火星木条复燃。

### 3.4 成功标准

- 学生在 5 秒内能看出当前实验正在发生什么现象。
- 每个实验至少有 3 个可区分的视觉状态。
- 视觉现象旁边必须有简短观察提示，避免学生只看动画不理解含义。
- 移动端和桌面端都不应出现布局溢出或遮挡。

## 4. 记忆巩固设计

### 4.1 记忆卡片

每次生成实验反馈后，系统同步生成一张实验记忆卡片。卡片包含：

- 实验名称
- 关键现象
- 化学方程式
- 核心原理
- 易错提醒
- 安全提醒
- 记忆口诀或一句话抓手
- 掌握状态：新学、需复习、已掌握

记忆卡片不是新知识库，而是从现有实验规则库和本次智能体反馈中提炼出来的学习材料。

### 4.2 三类回忆练习

V2 先实现三种轻量练习：

1. 遮答案回忆

   先展示问题，例如“CO2 通入澄清石灰水会出现什么现象”，学生点击后再显示答案。

2. 易错判断

   展示常见错误说法，例如“盐酸和氢氧化钠中和一定会产生大量气泡”，学生判断对错，再看解释。

3. 短问答

   展示一个简短问题，例如“铁放入硫酸铜溶液后，为什么铁表面会出现红色物质”，学生先口头或文字回忆，再展开参考答案。

### 4.3 今日复习

页面新增“今日复习”区域，根据卡片状态提示学生复习：

- 新学：刚完成实验，需要第一次回看。
- 需复习：答错、未回忆或距离上次学习较久。
- 已掌握：最近回忆正确，可暂时降低优先级。

第一版规则可以保持简单：

- 新生成的卡片默认为“新学”。
- 学生完成一次回忆后，如果选择“记住了”，状态变为“已掌握”。
- 如果选择“还不熟”，状态变为“需复习”。
- localStorage 保存卡片状态，刷新页面后仍能看到复习任务。

## 5. 页面结构调整

V2 首屏仍然是实验工作台，不做宣传页。

推荐布局：

- 左侧：实验选择、学生预期输入、实验播放控制。
- 中间：视觉化实验舞台和步骤时间线。
- 右侧：智能体解释、方程式、结论、安全提醒。
- 下方：学习轨迹、记忆卡片、今日复习。

在移动端，采用纵向顺序：

1. 实验选择与输入
2. 视觉化实验舞台
3. 智能体解释
4. 记忆卡片与今日复习
5. 学习轨迹与项目反馈

## 6. 数据与模块边界

### 6.1 Domain 层

新增或扩展以下领域能力：

- visualTimeline：根据实验返回可视化步骤和状态。
- memoryCard：根据实验和反馈生成记忆卡片。
- reviewScheduler：根据卡片状态生成今日复习列表。

这些逻辑应保持纯函数，便于单元测试。

### 6.2 Component 层

新增或改造以下组件：

- ExperimentStage：从静态双试管升级为可播放实验舞台。
- ExperimentTimeline：展示步骤并支持点击切换。
- MemoryCardPanel：展示实验记忆卡片和遮答案回忆。
- ReviewPanel：展示今日复习、易错判断和短问答。

### 6.3 State 层

App 需要增加：

- 当前实验播放步骤。
- 当前实验记忆卡片。
- 本地保存的记忆卡片列表。
- 复习状态更新动作。

localStorage 只保存必要字段：实验 id、卡片内容、掌握状态、创建时间、最近复习时间。

## 7. 测试策略

V2 需要补充以下测试：

- visualTimeline 能为 4 个实验生成正确步骤和视觉状态。
- memoryCard 能从实验规则和反馈中生成现象、方程式、原理、易错点、安全点。
- reviewScheduler 能根据新学、需复习、已掌握状态排序复习任务。
- 用户生成实验反馈后，会同步生成记忆卡片。
- localStorage 读写异常时，页面仍能正常使用，只是不保留长期复习状态。

浏览器验证重点：

- 桌面端实验舞台无遮挡，动画状态可见。
- 移动端无横向滚动，按钮和卡片文字不挤压。
- 复习卡片可展开/收起，状态切换后今日复习同步更新。

## 8. 暂不纳入 V2 的内容

以下方向放入后续版本：

- 真实 3D/VR 实验室。
- AI 自动生成实验视频。
- 账号系统和云端数据库。
- 教师端班级错题统计。
- 完整间隔重复算法和长期学习画像。
- 接入真实大模型 API 进行开放式问答。

这些方向有价值，但会增加实现不确定性。V2 先把可展示、可解释、可记忆的闭环做扎实。

## 9. V2 完成后的比赛表达

V2 完成后，作品报告可以把亮点升级为：

- 从“实验解释平台”升级为“实验学习闭环平台”。
- 不只回答学生问题，还帮助学生观察、解释、记忆和复习。
- 视觉化实验提升课堂展示性，记忆卡片与今日复习体现学生实验素养的持续迭代。
- 项目仍保持本地可复现，便于评委现场打开浏览器验证核心功能。
